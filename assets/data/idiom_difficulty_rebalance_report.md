# 成语难度重平衡报告

- 源文件：`idiom_crossword.db`
- 输出文件：`idiom_crossword_rebalanced.db`
- 总数：29,502
- SQLite integrity_check：`ok`
- 处理方法：`monotonic_quantile_v1`

## 字段变更

- `difficulty`：已覆盖为新的 1-50 相对游戏难度。
- `difficulty_original`：保留原始 LLM 难度。
- `difficulty_rank`：全局难度排名，1 最简单，29502 最难。
- `difficulty_percentile`：全局难度百分位，0-1。
- `bucket_rank_score`：原始分数桶内相对难度，0-100。
- `difficulty_feature_score`：桶内排序所用综合特征分。
- `difficulty_method`：处理方法标记。

## 方法说明

先保留原始 LLM 分数的单调顺序；同一个原始分数桶内，使用以下特征做重排：

1. 成语字符在题库中的稀有度。
2. 释义长度。
3. 释义中是否出现古代、原指、典故、亦作、同某成语等冷门/变体提示。
4. 拼音缩写稀有度。
5. 极小的确定性哈希扰动，仅用于稳定打破完全并列。

重排后按全局排名映射到 50 个等量难度档，每档约 590 条。
