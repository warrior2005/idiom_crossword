# 成语填字游戏 — 科举仕途成长系统设计

> 设计文档 | 2026-07-04 | v1.0

---

## 一、设计概述

### 1.1 目标

将成语填字游戏从"纯关卡挑战"升级为"成长驱动型"产品，通过科举仕途主题的等级体系、螺旋难度递增和道具系统，为玩家提供长期游玩目标。

### 1.2 核心设计原则

- **成长感**：20 级指数经验曲线，前期快速晋升，后期长线留存
- **自然过渡**：螺旋难度模型，成语 1-50 难度粒度，避免难度断层
- **收藏价值**：所有已完成成语自动收录，附带释义出处
- **内购平衡**：功能道具+装饰道具，不影响游戏难度和成语选择

### 2.2 经验公式

```
XP(n) = 100 × 1.6^(n-1)
```

- 第 1 级→第 2 级：100 经验（约 20 关）
- 第 5 级→第 6 级：1,500 经验（约 150 关）
- 第 10 级→第 11 级：16,000 经验（约 800 关）
- 第 15 级→第 16 级：160,000 经验（约 4,000 关）
- 第 19 级→第 20 级：1,000,000 经验（约 10,000 关）

### 2.3 经验获取

- 每通关一关获得经验 = 该关成语难度均值 × 2
- 教学关（1-5关）经验固定为 10
- 无其他经验来源（纯通关驱动）

## 三、关卡结构

### 3.1 关卡总量

- **总关卡**：10,000+ 关
- **教学关**（1-5关）：每关 5 条成语，纯难度 1
- **正式关**（6关+）：每关 8-12 条成语（波动范围待 UI 测试后确定）

### 3.2 成语池

- **总量**：29,502 条四字成语（已评分 1-50）
- **保底规则**：每条成语至少出现 3 次
- **复用策略**：全局池随机抽取，按难度权重选择

### 3.3 难度映射

关卡基准难度 = ceil(关卡编号 / 200)  （线性映射到 1-50）

## 四、螺旋难度模型

### 4.1 核心理念

难度递增采用"螺旋上升"模式，而非严格隔离：
- 每关以当前难度区间的成语为主体（60-70%）
- 混入前一难度区间的"长尾"成语（10-20%），保持连贯感
- 偶尔引入下一难度区间的"预览"成语（5-10%），制造新鲜感

### 4.2 难度分布示意

```
难度1-5   ████████████████████░░░░░░░░░░░░░░░░  ← 前500关主力
难度6-10  ░░░░░░████████████████████░░░░░░░░░░  ← 300-2000关主力
难度11-20 ░░░░░░░░░░░░░░████████████████████░░  ← 1500-4000关主力
难度21-30 ░░░░░░░░░░░░░░░░░░░░░░████████████░░  ← 3500-7000关主力
难度31-40 ░░░░░░░░░░░░░░░░░░░░░░░░░░░░████████  ← 6000-9000关主力
难度41-50 ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░████  ← 8500+关主力
          1    1000  2000  3000  4000  5000  6000+
                       关卡编号
```

### 4.3 选词规则

每关选词逻辑：
1. 计算基准难度 = ceil(关卡编号 / 200)
2. 主体选词（6-7 条）：从基准难度±3 范围内随机选取
3. 长尾散落（1-2 条）：从比基准低 5-10 的难度中随机选取
4. 预览混入（0-1 条）：从比基准高 3-5 的难度中随机选取

### 4.4 示例

| 关卡 | 基准难度 | 成语构成 |
|------|----------|----------|
| 第 100 关 | 1 | 难度1×5（纯教学） |
| 第 500 关 | 3 | 难度1×2 + 难度3×4 + 难度5×0 |
| 第 1000 关 | 5 | 难度1×1 + 难度4×3 + 难度6×3 + 难度8×1 |
| 第 3000 关 | 15 | 难度10×1 + 难度13×3 + 难度16×3 + 难度18×1 |
| 第 6000 关 | 30 | 难度25×1 + 难度28×3 + 难度31×3 + 难度33×1 |
| 第 9000 关 | 45 | 难度40×1 + 难度43×3 + 难度46×3 + 难度48×1 |

### 4.5 螺旋规则

- 难度 1 的成语在前 3000 关持续出现，占比逐渐降低
- 每个难度区间自然过渡，无断层
- 长尾成语保持游戏连贯感，避免"突然变难"

## 五、道具系统

### 5.1 功能道具

| 道具 | 效果 | 获取途径 |
|------|------|----------|
| 提示卡 | 揭示单个空格答案 | 等级奖励 / 内购 |
| 复活卡 | 失败后可继续当前关 | 等级奖励 / 内购 |

### 5.2 装饰道具

| 类型 | 示例 | 获取途径 |
|------|------|----------|
| 网格皮肤 | 竹简、宣纸、龙纹、金箔、九五至尊 | 等级解锁（基础）/ 内购（稀有） |
| 头像框 | 乌纱帽、獬豸冠、三公 | 等级解锁 / 内购 |
| 称号特效 | 金榜题名、天子门生 | 等级解锁 / 内购 |
| 自定义称号 | 位极人臣后可自定义 | 20 级解锁 |

### 5.3 内购设计

- **内购入口始终存在**，玩家可随时访问商城
- **功能道具包**：提示卡×10 / 复活卡×5 等组合包
- **装饰道具**：等级奖励以外的稀有皮肤、限定头像框、特效
- **不卖数值**：内购不影响关卡难度和成语选择

## 六、收藏系统（基础功能）

### 6.1 收录规则

### 6.2 筛选与搜索

- 可按难度/主题/收藏时间筛选

## 八、UI 适配要点

### 8.1 成语数量波动

- 教学关（1-5关）：固定 5 条成语
- 正式关（6关+）：8-12 条成语，波动范围待 UI 测试后确定
- 网格布局需支持动态行列数

### 8.2 等级展示

- 主界面显示当前等级称号和经验进度条
- 升级时弹出庆祝动画，展示奖励内容

### 8.3 商城入口

- 主界面常驻商城入口图标
- 商城内分"功能道具"和"装饰道具"两个 Tab

## 九、与现有模块的集成

### 9.1 IntegratedGenerator

- 接收关卡难度参数，从全局成语池中按螺旋规则选取成语
- 生成 CrosswordGrid 时需记录每条成语的难度，用于经验计算

### 9.2 DistractorEngine

- 无修改，继续为每个空格生成干扰字

### 9.3 Database Schema

- 新增 `player_progress` 表：等级、经验、道具数量
- 新增 `collection` 表：已收藏成语
- 新增 `level_history` 表：通关记录
- 新增 `decoration` 表：装饰道具拥有状态
